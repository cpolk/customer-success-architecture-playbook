# Sample Customer Adoption Plan

## Customer profile
A large enterprise with:
- multiple development teams
- inconsistent CI/CD patterns
- growing security requirements
- limited visibility into dependency risk
- long pull request cycle times

## Business priorities
- improve software delivery predictability
- reduce security and compliance risk
- increase developer productivity
- create repeatable engineering standards

## 90-day plan

### Days 1-30
- identify executive sponsor and technical stakeholders
- baseline current workflow metrics
- select pilot repositories
- define branch protection and pull request standards

### Days 31-60
- implement CI workflows for pilot repos
- enable dependency review and core security controls
- publish onboarding documentation
- review early outcomes with stakeholders

### Days 61-90
- expand to additional teams
- create champion network
- establish monthly value review
- produce executive readout with metrics, risks, and next-step recommendations
