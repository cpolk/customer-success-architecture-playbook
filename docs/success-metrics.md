# Success Metrics

## Principles
Good customer success metrics should be:
- measurable
- relevant to customer priorities
- understandable to executives and practitioners
- trendable over time

## Example metric categories

### Delivery Performance
- Deployment frequency
- Lead time for change
- Pull request cycle time
- CI success rate

### Security and Risk
- Mean time to remediate vulnerabilities
- Percentage of repositories with dependency review enabled
- Percentage of repositories with branch protection enabled
- Secret exposure incidents

### Adoption
- Active repositories using standard workflows
- Teams onboarded to target platform capabilities
- Percentage of developers using recommended workflows
- Time to onboard a new team

### Business Outcomes
- Reduced release delays
- Reduced manual approval effort
- Improved audit readiness
- Improved developer satisfaction
