# Adoption Framework

## 1. Discover
Goal: Understand business priorities, engineering friction, security requirements, and organizational constraints.

Key questions:
- What software delivery problems is the customer trying to solve?
- What are the most important business outcomes?
- Where are the bottlenecks in the developer workflow?
- What compliance or security constraints shape the rollout?

Outputs:
- Stakeholder map
- Current-state architecture
- Risks and constraints
- Initial value hypothesis

## 2. Align
Goal: Translate customer priorities into a practical platform strategy.

Focus areas:
- source control governance
- pull request workflow standards
- CI/CD maturity
- dependency management
- security controls
- developer enablement

Outputs:
- target-state architecture
- phased rollout plan
- success metrics and baselines
- executive summary

## 3. Implement
Goal: Deliver quick wins while building trust and momentum.

Examples:
- standardize pull request templates
- implement CI checks
- enable dependency review
- document repository governance
- onboard pilot teams

Outputs:
- pilot completion report
- adoption blockers log
- enablement artifacts

## 4. Expand
Goal: Move from pilot success to organizational scale.

Examples:
- expand to more teams
- establish champions
- define reusable patterns
- align with security and platform engineering teams

Outputs:
- scaled rollout plan
- operating cadence
- KPI dashboard

## 5. Realize Value
Goal: Prove impact and support renewal/expansion conversations.

Measures may include:
- deployment frequency
- lead time for change
- pull request cycle time
- security remediation time
- developer onboarding time
- percentage of repositories meeting policy standards
